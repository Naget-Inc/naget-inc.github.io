NagetBot is the web crawler that builds the search index for Naget

Identity

If you see NagetBot in your server logs, it's our crawler fetching publicly accessible HTML to make it discoverable through Naget. Use the details below to identify legitimate requests.

NagetBot

naget.com

Purpose

NagetBot crawls the public web to build a search index for naget.com. It fetches HTML only — it does not execute JavaScript, and does not interact with forms, logins, or paywalled content.

robots.txt compliance

NagetBot follows the Robots Exclusion Protocol. Specifically:

  • Fetches /robots.txt over HTTPS before crawling and obeys Disallow rules for the NagetBot user-agent.
  • Honors the Crawl-delay directive (capped at 120 seconds).
  • Honors HTTP 429 Too Many Requests and the Retry-After header (capped at 30 seconds).
  • Caches robots.txt per-domain with a TTL.

To block NagetBot entirely:

User-agent: NagetBot
Disallow: /

Disallow specific paths:

User-agent: NagetBot
Disallow: /private/
Disallow: /admin/

Set a crawl delay (seconds between requests, max 120):

User-agent: NagetBot
Crawl-delay: 10

Request behavior

  • Protocol: HTTPS preferred; falls back to HTTP if HTTPS fails.
  • Methods: GET only.
  • Content types fetched: HTML / XHTML only — non-HTML responses are dropped without reading the body.
  • Max response body: 5 MiB (truncated beyond that).
  • Timeouts: ~2s connect, ~3s read by default.
  • Cookies / auth / Referer: none sent.

Headers sent with every request:

User-Agent: Mozilla/5.0 (compatible; NagetBot/1.0; +https://naget.ai/bot)
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
Accept-Language: en-US,en;q=0.9

Rate limiting

  • Per-domain concurrency: max 8 concurrent requests.
  • Minimum delay: 50 ms between requests to the same domain (overridden upward by any Crawl-delay directive).
  • Retries: one retry with backoff on transient errors or 429s, then the URL is dropped.
  • Error handling: domains returning repeated errors are skipped on subsequent recrawls.

Verifying NagetBot

Because the User-Agent string can be spoofed, we recommend verifying that requests genuinely originate from our infrastructure before granting bot-specific privileges.

A published list of source IP ranges and a reverse-DNS verification pattern will be available here shortly. If you need to verify a specific request in the meantime, please contact us using the channels below.

Page-level controls

You can control how individual pages are indexed using standard meta tags:

<meta name="NagetBot" content="noindex, nofollow">

Or apply directives to all crawlers:

<meta name="robots" content="noindex, nofollow">
